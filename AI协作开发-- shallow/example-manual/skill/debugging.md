# AI 调试技能

> **📎 何时读此文件**: 排查问题时、测试失败时、功能不正常时
> **对应原则**: DBG-01~DBG-03（Debug 相关原则）、AI-03（清理临时代码）
> **相关文件**: `rules/logging.md`, `../AgentHub-records/gotcha.md`

## 技能定义

**名称**: `debugging`
**描述**: 系统化调试技能，要求 AI 按照日志 → 定位 → 修复 → 验证的流程进行调试
**触发条件**: 任何功能不正常、测试失败、日志报错

---

## 调试流程

### 阶段 1：信息收集（必须先做）

1. **读取详细日志**：
   - 查找与该功能相关的所有日志
   - 特别关注 `ERROR`、`WARNING`、`DEBUG` 级别
   - 记录时间戳，确认日志顺序

2. **检查关键数据**：
   - 输入数据是否符合预期
   - 中间状态是否正确
   - 输出数据是否异常

3. **确认执行路径**：
   - 函数调用链是否正确
   - 分支逻辑是否按预期执行
   - 异步操作是否完成

### 阶段 2：问题定位

**定位方法优先级**：
1. **日志定位**（首选）：增加详细日志，重新执行，观察日志
2. **断点定位**（辅助）：在关键位置添加断点或调试输出
3. **排除法**（辅助）：注释部分代码，缩小问题范围
4. **对比法**（辅助）：与正常分支对比，找出差异

**禁止**：
- ❌ 不查看日志直接修改代码
- ❌ 只改一处就测试，不系统排查
- ❌ 用 `print` 调试，不用日志系统
- ❌ 忽略错误日志，只关注功能表现

### 阶段 3：修复

1. **理解根因**：必须明确为什么出错，而不是仅仅消除症状
2. **最小修改**：只修改必要的代码，避免大面积重构
3. **保持语义**：不得修改返回值语义、类型、空值表示
4. **记录变更**：记录修改了什么、为什么修改

### 阶段 4：验证

1. **单元测试**：确保相关测试通过
2. **集成测试**：确保完整链路跑通
3. **边界测试**：测试异常情况
4. **日志检查**：确认日志输出正常

**验证清单**：
- [ ] 功能是否恢复正常
- [ ] 日志是否详细且正确
- [ ] 测试是否全部通过
- [ ] 边界情况是否处理
- [ ] 是否有副作用
- [ ] 同类错误是否一并检查

## 调试日志模板

每次调试必须增加以下日志：

```python
# 函数入口
logger.info(f"[FUNCTION] {func_name} 开始执行，参数: {params}")

# 关键分支
logger.info(f"[BRANCH] 进入 {branch_name} 分支，条件: {condition}")

# 函数出口
logger.info(f"[FUNCTION] {func_name} 完成，结果: {result}")

# 错误处理
logger.error(f"[ERROR] {func_name} 失败: {e}", exc_info=True)
```

## 同类错误检查

修复一个 bug 后，必须检查：

1. **相同代码模式**：项目中是否有类似写法
2. **相同逻辑路径**：其他分支是否有相同问题
3. **相同调用方**：其他调用该函数的地方是否受影响
4. **相同数据源**：数据是否在其他地方也有问题

**方法**：
- 用 `grep` 搜索相似代码
- 用 `grep` 搜索被修改的函数名
- 检查所有调用方

## 调试效率规则

1. **禁止反复读同一文件**：一次读取至少 100 行，不得反复读取
2. **禁止读空文件**：确认文件有内容再读取
3. **先问再动手**：如果问题描述模糊，先列出 3-5 个澄清问题
4. **记录灵感**：调试过程中产生的新想法，记录到 `todo.md`

---

## 临时调试代码

所有用于调试的临时输出必须：

1. **包含标识**：`#TEMP_DEBUG#`
2. **使用日志**：`logger.debug(f"#TEMP_DEBUG# {var}")`
3. **任务完成后删除**：搜索 `#TEMP_DEBUG#` 并删除

**禁止**：
- ❌ 使用 `print()` 调试
- ❌ 遗留调试代码不删除
- ❌ 调试代码写入生产环境

---

## 失败降级验证

如果功能有失败降级机制（如 weasyprint → fpdf2），必须验证：

- [ ] 主方案失败时日志是否记录
- [ ] 降级方案是否成功执行
- [ ] 降级方案输出是否正确
- [ ] 降级方案日志是否记录
- [ ] 全部失败时错误是否抛出

---

> 💡 **提示**：调试不是试错。每次修改必须有明确的假设和验证。
